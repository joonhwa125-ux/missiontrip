# 구글 시트 데이터 입력 가이드

> `/setup` 화면에서 데이터를 동기화할 때 사용하는 구글 시트 구조 정의.
> 파서 구현: `src/utils/sheets-parser.ts`, 임포트 로직: `src/actions/setup.ts`

---

## 시트 구성 (2개)

| 시트명 | 용도 |
|---|---|
| 참가자 | 전체 참가자 목록 (조원, 조장, 관리자). 소속조 고유값에서 Groups 자동 생성 |
| 일정 | 2박3일 일정 목록. 예정시각 기반 자동 활성화에 사용 |

---

## 시트 1 — 참가자

### 컬럼 순서 (6개, 1행은 헤더)

| # | 컬럼명 | 필수 | 허용값 | 예시 |
|---|---|---|---|---|
| 1 | 이름 | ✅ | 텍스트 | 홍길동 |
| 2 | 전화번호 | - | 텍스트 (`010-XXXX-XXXX`) | 010-1234-5678 |
| 3 | 역할 | ✅ | `조원` / `조장` / `관리자` | 조장 |
| 4 | 소속조 | ✅ | 텍스트 | A조 |
| 5 | 배정차량 | - | 텍스트 | 1호차 |
| 6 | 선후발 | - | `선발` / `후발` | 선발 |

### 규칙

- **헤더 행:** 1행은 반드시 컬럼명 (파서가 자동 스킵)
- **이메일 자동 생성:**
  - 조장/관리자 → `{이름소문자}@linkagelab.co.kr`
  - 조원 → `_member.{이름}.{행번호}@nologin.internal` (로그인 불가, 식별용)
- **역할 괄호 표기:** `조장(관리자)` 형식 허용 → 괄호 안 값(`관리자`)이 우선 적용
- **소속조 → 그룹 자동 생성:** 소속조 컬럼의 고유값이 `groups` 테이블에 자동 INSERT
- **배정차량:** 비어있으면 소속조 이름을 차량명으로 사용
- **선후발:** 비어있으면 전체 일정에 참여 (`party = null`)
- **동명이인 조장/관리자:** 이메일 충돌 → `/setup` 미리보기에서 오류 표시

### 검증 오류 조건

| 조건 | 오류 메시지 |
|---|---|
| 이름 비어있음 | 이름이 없어요 |
| 역할값 불일치 | 역할은 조원/조장/관리자만 가능해요 |
| 소속조 비어있음 | 소속조가 없어요 |
| 선후발값 불일치 | 선후발은 선발/후발만 가능해요 |
| 조장/관리자 이메일 중복 | {이름} 이름이 중복되어 있어요 (이메일 충돌) |
| 조에 조장 없음 | {조이름}에 조장이 없어요 |

### 예시 데이터

```
이름, 전화번호, 역할, 소속조, 배정차량, 선후발
홍길동, 010-1234-5678, 조장, A조, 1호차, 선발
김철수, 010-9876-5432, 조원, A조, 1호차, 선발
이영희, 010-1111-2222, 관리자, 관리팀, ,
박민준, 010-3333-4444, 조원, B조, 2호차, 후발
최수진, 010-5555-6666, 조장, B조, 2호차, 후발
```

## 시트 2 — 일정

### 컬럼 순서 (6개, 1행은 헤더)

| # | 컬럼명 | 필수 | 허용값 | 예시 |
|---|---|---|---|---|
| 1 | 일차 | ✅ | `1` / `2` / `3` | 1 |
| 2 | 순서 | ✅ | 숫자 | 1 |
| 3 | 일정명 | ✅ | 텍스트 | 김포공항 탑승 게이트 집결 |
| 4 | 장소 | - | 텍스트 | 국내선 3번 게이트 |
| 5 | 예정시각 | - | 아래 형식 참조 | `06-15 08:30` |
| 6 | 대상 | - | `전체` / `선발` / `후발` | 선발 |

### 예정시각 형식

자동 활성화 기능이 `scheduled_time`을 기준으로 동작하므로 **날짜 포함 형식 권장**.

| 형식 | 예시 | 동작 | 권장 |
|---|---|---|---|
| `MM-DD HH:MM` | `06-15 08:30` | 현재 연도 + 지정 날짜 | ✅ 권장 |
| `YYYY-MM-DD HH:MM` | `2026-06-15 08:30` | 연도까지 명시 | 선택 |
| `HH:MM` | `08:30` | 임포트한 당일 날짜 사용 | ⚠ 비권장 |

> `HH:MM`만 입력하면 데이터 임포트 날짜가 기준이 됨. 행사일 이전에 임포트한 경우
> 자동 활성화가 오작동할 수 있으므로 `MM-DD HH:MM` 형식 사용 권장.

### 규칙

- **순서(`sort_order`):** 같은 일차 내 카드 표시 순서. 숫자가 작을수록 위에 표시
- **대상:** 비어있으면 `전체`로 처리. 선발/후발 전용 일정은 해당 party 인원만 카운트
- **장소:** 체크인 화면 상단에 굵게 표시 (조장이 어디에 있어야 하는지 최우선 정보)

### 검증 오류 조건

| 조건 | 오류 메시지 |
|---|---|
| 일차 범위 벗어남 | 일차는 1~3만 가능해요 |
| 일정명 비어있음 | 일정명이 없어요 |
| 대상값 불일치 | 대상은 전체/선발/후발만 가능해요 |

### 예시 데이터

```
일차, 순서, 일정명, 장소, 예정시각, 대상
1, 1, 김포공항 탑승 게이트 집결, 국내선 3번 게이트, 06-15 08:30, 선발
1, 2, 제주공항 도착 인원 확인, 1층 출구 앞, 06-15 10:00, 선발
1, 3, 숙소 체크인, 숙소 로비, 06-15 15:00, 전체
2, 1, 버스 탑승 (서귀포 이동), 숙소 앞, 06-16 09:00, 전체
2, 2, 점심 식당 집결, 서귀포 식당 앞, 06-16 12:00, 전체
3, 1, 제주공항 출발 집결, 2층 출발장, 06-17 13:00, 전체
```

## 구글 시트 공유 설정

`/setup`에서 Google Sheet URL로 불러오는 경우:

1. 시트 우측 상단 **[공유]** 클릭
2. **"링크가 있는 모든 사용자"** 선택
3. 권한: **"뷰어"** 설정
4. `/setup` → Google Sheet URL 입력 → 참가자/일정 시트 GID 각각 입력

> GID 확인: 시트 탭 클릭 후 URL에서 `gid=XXXXXXX` 값 복사

## 재동기화 (UPSERT)

동일 데이터를 다시 임포트해도 안전하게 덮어씀:

| 테이블 | 충돌 기준 |
|---|---|
| groups | `name` |
| users | `email` |
| schedules | `day_number, sort_order` |

참가자 변경, 예정시각 수정 등은 재동기화로 반영 가능.
