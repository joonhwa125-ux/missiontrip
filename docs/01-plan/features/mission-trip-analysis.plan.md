### 1.3 핵심 갭 분석 — 우선도 높은 항목

#### GAP-01: 조장 부재 시 대행자 지정 불가 (E01)
- **상황**: 조장이 몸이 아프거나 비행기 좌석 분리 등으로 조원 옆에 없음
- **현재 대응**: 없음. 해당 조 체크인이 전원 미완료 상태로 방치됨
- **권장 대응**: TF장(admin)이 모든 조의 체크인 대행 가능 → 이미 DB RLS에서 admin 허용. 다만 UI에서 TF장이 특정 조를 선택해 체크인하는 경로가 명시되지 않음
- **중요도**: 높음

#### GAP-02: 불참 사유 구분 불가 (E04, E06, E07)
- **상황**: 불참 처리 시 '자비 이동', '의료응급', '중도귀가', '비행기 분리 탑승' 구분 불가
- **현재 대응**: 미탑승/불참 처리는 가능하나 사유 메모 없음
- **권장 대응**: 간단한 사유 선택지 또는 자유 메모 필드 (선택사항)
- **중요도**: 중간 (안전에는 직접 영향 없음)

#### GAP-03: 조원 임시 조 이동 처리 불가 (E02)
- **상황**: A조 조원이 특별 사정으로 B버스를 타야 할 때 B조 조장이 체크인 불가
- **현재 대응**: 없음. 타 조 조원에 대한 체크인 권한 없음 (group_id 기반 RLS)
- **권장 대응**: TF장이 임시 그룹 변경 또는 조장이 자기 조 외 인원 체크인 허용 옵션
- **중요도**: 중간

### 3.2 핵심 갭: 조장 연락처 접근성

기존 plan 문서(mission-trip-v1)에 "조장 연락처" 언급이 있으나 UI 명세에 구체적 위치가 불명확합니다.

- **현재**: 조 카드 드릴다운 → 미탑승 인원 이름+전화번호 표시
- **부재**: 조장 본인의 전화번호를 카드 겉면에서 바로 볼 수 없음
- **권장**: 조 카드(그리드 뷰)에 조장 이름+전화번호 `010-xxxx-xxxx` 형식으로 노출

## 5. 유저 스토리 검증

### 5.1 핵심 유저 스토리 목록

#### 조장 관련

**US-01: 진행중 일정에서 조원 체크인**
```
조장으로서, 현재 진행중인 일정을 피드에서 바로 확인하고
조원 한 명씩 [탔어요!] 버튼을 탭하여 체크인을 처리할 수 있다.
```

| 수락 기준 | 현재 기획 충족 여부 |
|-----------|-------------------|
| 피드에서 진행중 일정이 강조 표시됨 | 충족 — 노란 배경 + 진행중 pill |
| 탭 1회로 체크인 드릴다운 진입 | 충족 |
| [탔어요!] 탭 시 확인 없이 즉시 체크인 | 충족 |
| 완료 카드가 하단으로 자동 이동 | 충족 |
| 오프라인 시 로컬 저장 후 나중에 sync | 충족 |

결과: **완전 충족**

**US-02: 조원 불참 처리**
```
조장으로서, 특정 조원이 이번 이동에 탑승하지 않음을 기록할 수 있다.
(자비 이동, 체력 문제 등)
```

| 수락 기준 | 현재 기획 충족 여부 |
|-----------|-------------------|
| 불참 처리 버튼이 존재함 | 충족 (CLAUDE.md에 불참 처리 명시) |
| 불참 사유를 선택/입력할 수 있음 | **미충족** — 사유 기록 기능 없음 |
| 불참 처리된 조원이 시각적으로 구분됨 | 미명세 — 색상/아이콘 구분 명세 필요 |
| 불참 처리 후 보고 시 TF장에게 불참 인원 수 전달됨 | 충족 — pending_count에 포함 |

결과: **부분 충족** (사유 기록 없음)

```
조장으로서, 우리 조 모든 조원의 탑승을 확인한 후
TF장에게 보고할 수 있다.
```

| 수락 기준 | 현재 기획 충족 여부 |

| 미완료 시에도 보고 가능 (disabled 금지) | 충족 |
| 미완료 시 확인 모달 표시 | 충족 |
| 보고 시 TF장 화면에 실시간 반영 | 충족 — Realtime broadcast |
| 보고 내용이 DB에 영속화됨 | 충족 — group_reports |

결과: **실질적 충족** (축하 화면은 선택사항)

| TF장이 대행 체크인 처리 기록이 남음 | 충족 — checked_by='admin' |

결과: **부분 충족** (UI 경로 명확화 필요)

## 6. 종합 권장사항 (MoSCoW)

### Must (현재 기획에 반드시 보완)

| # | 항목 | 근거 |
|---|------|------|
| M-01 | 관리자 화면 조 카드에 **조장 전화번호** 겉면 노출 | 긴급 연락이 가장 빈번한 행동, 드릴다운 없이 즉시 전화 가능해야 함 |
| M-02 | **진행중 일정 없음** 빈 상태 UI 명세 추가 | 조장이 앱 접속 시 아무것도 없으면 당황함 |
| M-03 | **TF장의 타 조 체크인 대행** UI 경로 명시 | 조장 부재 시 안전 보장을 위한 폴백 경로 |

### Should (기획에 포함하면 좋음)

| # | 항목 | 근거 |
|---|------|------|
| S-01 | **불참 처리 시 간단 사유** 선택지 (자비이동/의료/기타) | 운영 후 보고서 작성 시 유용, 구현 비용 낮음 |
| S-02 | **앱 진입 시 활성 일정 자동 포커스** (피드 최상단 진행중 일정 강조) | 조장 화면 전환 비용 감소 |
| S-03 | **전원 완료 축하 화면** 1차 포함 검토 | 2박3일 피로 누적 환경에서 심리적 보상 중요 |

### Could (여유 있으면 고려)

| # | 항목 | 근거 |
|---|------|------|
| C-01 | 불참 처리된 조원의 이후 일정 자동 불참 옵션 | 수동 처리 반복 부담 감소, 단 잘못된 자동화 위험 있음 |
| C-02 | 관리자 화면 전체 미탑승 인원 수 요약 | 출발 결정 시 즉시 파악 |
| C-03 | 이니셜 동그라미 탭 → 해당 조원 카드 자동 스크롤 | 20명 이상 조에서 편의성 향상 |

### Won't (이번 버전 제외)

| # | 항목 | 이유 |
|---|------|------|
| W-01 | 조원 임시 조 이동 기능 | 복잡도 대비 발생 빈도 낮음, TF장 대행으로 대응 |
| W-02 | 조원 셀프 체크인 | 이미 제거 결정 |
| W-03 | 공지 기능 | 카카오워크 대체 결정 |

## 8. 결론

현재 기획은 핵심 기능(체크인 탭탭탭, Realtime 동기화, 오프라인 대응, 일정 관리)의 완성도가 높습니다.

보완이 필요한 3가지 Must 항목을 처리하면 현장 운영에 충분한 수준이 됩니다:

1. **조장 전화번호를 관리자 화면 카드 겉면에 노출** — 긴급 상황 대응 속도 직결
2. **진행중 일정 없음 빈 상태 UI** — 조장 혼란 방지
3. **TF장 대행 체크인 UI 경로 명시** — 조장 부재 시 안전망 확보
