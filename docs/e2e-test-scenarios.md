# E2E 테스트 시나리오

> 운영 전 수동 검증용. 브라우저 2개(관리자 A, 조장 B)로 진행.
> 각 시나리오의 체크박스([ ])를 통과 시 [x]로 표시.

---

- [ ] Vercel 프로덕션 배포 완료
- [ ] 테스트용 데이터 셋업 완료 (/setup에서 Google Sheets 또는 CSV import)
- [ ] 브라우저 A: 관리자 계정 로그인 → /admin
- [ ] 브라우저 B: 조장 계정 로그인 → /group
- [ ] 두 브라우저 모두 같은 조에 소속된 조장이 아닌, **관리자 vs 조장** 구성

## 1. 로그인 흐름

### 1-1. 정상 로그인

- [ ] 로그인 후 새로고침 → 세션 유지 (재로그인 요구 없음)

### 1-2. 비정상 로그인
- [ ] 회사 외 구글 계정으로 시도 → '회사 계정으로만 로그인할 수 있어요' 안내

## 2. 일정 활성화 (관리자 A)

### 2-1. 수동 활성화
- [ ] A: 대기 일정 카드의 [활성화] 버튼 탭
- [ ] A: 해당 카드가 '진행중' pill + 노란 테두리로 변경
- [ ] B: 실시간으로 '새로운 일정이 시작되었어요: {일정명}' 토스트 수신
- [ ] B: 일정 피드에서 해당 일정이 '진행중'으로 표시

### 2-2. 미확인 경고
- [ ] A: 활성 일정에 미확인 인원 있는 상태에서 다른 일정 [활성화] 시도
- [ ] A: '현재 일정에 N명이 미확인 상태예요. 그래도 전환할까요?' 경고 모달 표시
- [ ] A: [취소] → 기존 일정 유지
- [ ] A: [확인] → 새 일정으로 전환

### 2-3. 일정 종료
- [ ] A: 진행중 일정을 종료 (비활성화)
- [ ] B: '일정이 종료되었어요' 토스트 수신
- [ ] A: 해당 카드가 완료(dim) 상태로 변경

## 3. 조장 체크인 — 탭탭탭 핵심 흐름 (조장 B)

### 3-1. 체크인 처리
- [ ] B: 진행중 일정 카드 탭 → 체크인 화면 진입
- [ ] B: 상단에 '{조명}' + 집결장소 표시 확인
- [ ] B: 미완료 조원 카드의 [왔수다!] 탭 → **모달 없이 즉시** 완료 처리
- [ ] B: 카드가 연초록 배경 + 'HH:MM 탑승 완료' + 초록 체크 배지로 변경
- [ ] B: 완료 카드가 하단으로 자동 이동
- [ ] B: 이니셜 원이 노란 배경 + 초록 체크 점으로 변경
- [ ] B: 상단 카운트 증가 (N/M명 완료)

### 3-2. 체크인 취소
- [ ] B: 완료 카드의 [취소] 탭 → '체크인을 취소할까요?' 모달
- [ ] B: [확인] → 미완료 상태로 복귀 (흰 배경, '확인 전')
- [ ] B: 카운트 감소

### 3-3. 불참 처리
- [ ] B: 미완료 카드의 [불참] 탭 → '불참 처리할까요?' 모달
- [ ] B: [확인] → 회색 배경 + '− 불참' 앰버 배지
- [ ] B: 이니셜 원에 마이너스(−) 아이콘 표시
- [ ] B: 탑승 카운트에서 제외되지만 완료 카운트에는 포함

### 3-4. 불참 취소
- [ ] B: 불참 카드의 [취소] 탭 → '불참을 취소할까요?' 모달
- [ ] B: [확인] → 미완료 상태로 복귀

## 4. 전원 완료 + 보고 (조장 B)

### 4-1. 전원 확인 완료
- [ ] B: 모든 조원 체크인 또는 불참 처리 완료
- [ ] B: 축하 화면 전환 — standing-horse 이미지(정지) + '전원 확인 끝!' + '보고하기를 눌러주세요.'
- [ ] B: 하늘색 그라데이션 배경

### 4-2. 보고 전 비활성 상태
- [ ] B: 전원 확인 전에 보고 버튼이 'N명 남았어요' 텍스트 + aria-disabled 상태
- [ ] B: 비활성 상태에서 탭 → 아무 동작 없음

### 4-3. 보고 실행
- [ ] B: 전원 확인 후 [우리 조 다 왔수다! 보고하기] 탭 → **모달 없이 즉시 전송**
- [ ] B: '보고 완료!' 토스트 표시 (5초간)
- [ ] B: 축하 화면 변경 — running-horse 이미지(바운스) + '보고 완료!' + '폭싹 속았수다!'
- [ ] B: 보고 버튼 → '보고 완료!' 텍스트로 변경

### 4-4. 관리자 수신 확인
- [ ] A: 바텀시트에서 해당 조 배지가 '보고완료'로 실시간 변경
- [ ] A: 모든 조 보고 완료 시 '모든 조가 보고했어요' 표시

## 5. 관리자 현황 + 드릴다운 (관리자 A)

### 5-1. 바텀시트 현황
- [ ] A: 진행중 카드의 [전체 현황 보기] 탭 → 바텀시트 열림
- [ ] A: 차량(bus_name)별로 조가 그룹핑되어 표시
- [ ] A: 각 조 카드에 프로그레스 바 + 상태 배지 + 조장 이름 + 전화 아이콘
- [ ] A: 전화 아이콘 탭 → 전화 앱 연결 (tel: 링크)

### 5-2. 드릴다운 체크인 대행
- [ ] A: 바텀시트에서 조 카드 탭 → 해당 조 인원 목록
- [ ] A: 미완료 인원 옆 [왔수다!] 탭 → 체크인 처리
- [ ] B: 해당 조원 카드가 실시간으로 완료 상태로 변경

### 5-3. 드릴다운 불참 처리
- [ ] A: 드릴다운에서 미완료 인원 옆 [불참] 탭 → 확인 모달 → 불참 처리
- [ ] B: 해당 조원 카드가 실시간으로 불참 상태로 변경

### 5-4. 조장 권한 변경
- [ ] A: 드릴다운에서 조원 이름 옆 [조장 지정] 탭 → 확인 모달 → 역할 변경
- [ ] A: 해당 인원에 '조장' 배지 표시
- [ ] A: [조장 해제] 탭 → 확인 모달 → 조원으로 복귀

## 6. 실시간 동기화 (양방향)

### 6-1. 조장 → 관리자
- [ ] B: 체크인 처리 → A의 바텀시트 카운트 실시간 증가
- [ ] B: 보고 → A의 배지 '보고완료' 실시간 변경

### 6-2. 관리자 → 조장
- [ ] A: 일정 활성화 → B에 토스트 + 피드 갱신
- [ ] A: 드릴다운에서 체크인 대행 → B의 카드 실시간 변경
- [ ] A: 일정 시간 수정 → B에 '일정 시간이 변경되었어요' 토스트

### 6-3. 체크인 취소 → 보고 무효화
- [ ] B: 보고 완료 상태에서 한 명 체크인 취소
- [ ] B: 보고 상태가 자동 해제 (보고 버튼 재활성화)
- [ ] A: 해당 조 배지가 '보고완료' → '진행중'으로 변경

## 7. 일정 관리 (관리자 A)

### 7-1. 일정 추가
- [ ] A: [+ 일정 추가] 탭 → 일정명(필수), 장소, 일차, 예정시각 입력
- [ ] A: 저장 → 해당 일차에 새 카드 추가
- [ ] B: 새로고침 시 새 일정 표시

### 7-2. 일정 시간 수정
- [ ] A: 바텀시트에서 예정시각 수정 아이콘 탭 → 시:분 변경 → 저장
- [ ] B: '일정 시간이 변경되었어요' 토스트 수신

### 7-3. 일차 탭 전환
- [ ] A, B: [1일차] [2일차] [3일차] 탭 전환 → 해당 일차 일정만 표시
- [ ] 활성 일정의 일차가 기본 선택되는지 확인

- [ ] B: [왔수다!] 탭 → 낙관적 업데이트로 즉시 완료 표시

### 8-3. 오프라인 보고
