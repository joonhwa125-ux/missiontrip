## 미션트립 PRD 적합성 검증

CLAUDE.md(PRD)를 읽고, 현재 구현된 코드가 명세와 일치하는지 검증하라.

### 검증 항목

1. **스키마 일치**: DB 테이블 컬럼이 CLAUDE.md 섹션 3과 일치하는가
2. **화면 명세 일치**: 컴포넌트가 섹션 4의 UI 스펙(색상, 문구, 카드 높이 등)과 일치하는가
3. **절대 금지 사항**: line-through, FEE500 on 공지, 오프라인 배너 top, polling, 보고 disabled
4. **Realtime 채널**: 섹션 5의 채널/이벤트 구조와 일치하는가
5. **Server Actions**: 섹션 9의 함수 시그니처와 권한이 일치하는가
6. **컬러 시스템**: 섹션 8.1의 컬러값만 사용되었는가
7. **UI 문구**: 섹션 8.2의 문구만 사용되었는가

### 검증 절차

1. CLAUDE.md 전체를 읽는다
2. `src/` 디렉토리의 모든 .ts/.tsx 파일을 탐색한다
3. 각 검증 항목에 대해 위반 사항을 수집한다
4. 결과를 아래 형식으로 보고한다:

### 출력 형식

```
## PRD 적합성 검증 결과

### 일치 (N건)
- [항목]: 설명

### 불일치 (N건)
- [항목] 파일명:라인 — 명세: "..." vs 구현: "..."

### 미구현 (N건)
- [항목]: 아직 구현되지 않은 명세
```
